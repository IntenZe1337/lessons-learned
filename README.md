# lessons-learned

Lärdomar från incidents, felbeslut och nära-ögat-situationer i infrastruktur och kod.

## Syfte

Samlar konkreta lärdomar — vad som gick fel, varför, och vad som ändrades — så att samma misstag inte upprepas. Fokus på handlingsbara slutsatser snarare än kronologisk logg.

## Struktur

Varje lärdom är ett eget Markdown-dokument med:
- **Datum** och kort rubrik
- **Vad som hände** (fakta)
- **Rotorsak**
- **Åtgärd och förändring**
